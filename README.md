# prdx.so-laws

Контент правового свода prdx.so. Рендерится сайтом `plasmo-rw-frontend` через общий wiki-движок
(источник `WIKI_SOURCES.laws`, роут `/laws`). Этот репозиторий — **чисто контентный**: `pages/` + `assets/`.
Своего фронтенда здесь нет.

## Структура

```
pages/
  100-pamyatki/                 # раздел
    index.mdx                   # посадочная страница раздела (title в frontmatter)
    100-deystvuyushchee-zakonodatelstvo.mdx
    200-nalogi-i-sbory.mdx
  200-konstituciya/
    index.mdx
    100-glava-1.mdx ... 600-glava-6.mdx
  300-criminal-code/            # УК — 3 уровня: кодекс / раздел / глава
    index.mdx
    100-razdel-1/
      index.mdx
      100-glava-1.mdx ...
  ...
assets/                         # картинки, ссылки вида /assets/<file>
```

## Правила для редакторов (контракт движка)

- Контент только в `pages/`. Файлы только `.mdx` (`.md` движок игнорирует).
- Порядок задаёт числовой префикс `NNN-` в имени папки/файла (шаг 100). Без префикса → конец списка.
- URL-слуг = имя без префикса, поэтому имена **латиницей** (`100-glava-1` → `/laws/.../glava-1`).
- Человекочитаемый заголовок — в YAML-frontmatter `title` (кириллица допустима). Frontmatter обязателен
  и идёт первым блоком файла:
  ```
  ---
  title: "Глава 1. Конституционный строй"
  ---
  ```
- У каждой папки-раздела обязателен `index.mdx` с `title`, иначе её страница пустая.
- Глубина навигации — максимум 3 уровня (раздел → подраздел → страница). Глубже движок не строит.
- Картинки: `![alt](/assets/<file>)`. Резолвятся движком автоматически.
- Это **MDX**: голые `<` и `{` ломают сборку. В тексте `<` → `&lt;`, `{` → `&#123;`,
  угловые плейсхолдеры (`<ник>`) запрещены — пиши `[ник]` или экранируй.

## Локальное превью

В `plasmo-rw-frontend` задать `LAWS_LOCAL_PATH=/путь/к/этому/репо` и открыть `/laws`
(локальный режим обходит гейт доступа `staff`).
